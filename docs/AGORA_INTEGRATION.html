§1 · Agora Products Used

The extension uses four distinct Agora products. Each operates in a different runtime and serves a specific role:

ConvoAI
Conversational AI
Orchestrates the full STT→LLM→TTS pipeline. Hosts the AI agent in Agora's cloud. Called from the Node.js extension host via REST.
agora-agents SDK · npm · Node.js only
RTC
Real-Time Communication
Carries the audio channel: mic capture from the browser companion, and agent voice playback. Browser API — requires a real browser tab.
AgoraRTC_N.js · browser companion only
RTM
Real-Time Messaging
Delivers structured transcript events (user.transcription / assistant.transcription) to both the VS Code sidebar and the browser companion.
agora-rtm.js · sidebar + companion
Token
Token Builder
Generates RTC (AccessToken2) and RTM (AccessToken v1) tokens locally in Node.js. No network call — uses appId + appCertificate to sign tokens.
agora-token · npm · Node.js only

§3 · Authentication

Two auth modes are supported for the ConvoAI REST API. The mode is chosen automatically at session start:

Token Auth (default)
Uses appId + appCertificate only. The agora-agents SDK derives a ConvoAI token internally. No additional credentials needed.
Basic Auth (preferred)
Requires customerId + customerSecret from Agora Console → RESTful API keys. Sent as Authorization: Basic base64(id:secret) on every ConvoAI request.

Security note: The browser companion receives a pre-computed stopAuth value (base64 of id:secret) so it can call the stop endpoint on page unload, but appCertificate is never sent to any browser context.

§5 · Session Modes

Cascading Voice
Three swappable providers: STT → LLM → TTS. Audio over RTC. Transcripts via RTM. Mic in browser companion.

idleTimeout: 30
Realtime Speech
Single MLLM (e.g. OpenAI Realtime API or Gemini Live) handles STT+LLM+TTS internally. Same RTC+RTM transport.

idleTimeout: 30
Text Chat
No mic, no audio. User types in sidebar. session.think() injects text into LLM. Responses via RTM. No browser companion.

idleTimeout: 0

In all modes, the Agent is configured with advancedFeatures: { enable_rtm: true } and parameters: { data_channel: 'rtm', enable_error_message: true, enable_metrics: true }.

§9 · Token Builder APIs (agora-token — local, no network)

Called once during startSession() after the ConvoAI agent is created. Both calls are synchronous and local.

Token TypeBuilder CallFormatExpires
RTC Client Token RtcTokenBuilder.buildTokenWithRtm(appId, cert, channel, "1", RtcRole.PUBLISHER, 3600, 3600) AccessToken2 (includes RTM privileges) 3600 seconds (duration)
RTM Token RtmTokenBuilder.buildToken(appId, cert, "1", Math.floor(Date.now()/1000) + 3600) AccessToken v1 Absolute Unix timestamp (+3600s)
RTM token generation is wrapped in a try/catch — if it fails, the sidebar falls back to receiving transcripts via the RTC stream-message data channel instead.

§11 · Agora RTM Web SDK APIs (Sidebar + Companion)

RTM is used in both the webview sidebar and the browser companion. The webview uses RTM as its only Agora connection — no RTC. Both use the same client UID (1) and channel.

WhenAPI CallDescription
Session live new AgoraRTM.RTM(appId, String(clientUid)) Creates RTM client instance with UID="1"
Session live rtmClient.login({ token: rtmToken }) Authenticates with RTM service using the minted AccessToken v1
Session live rtmClient.subscribe(channel) Subscribes to the session channel to receive agent transcript events

RTM Message Payload Format

Messages arrive as either a string or Uint8Array (decoded with TextDecoder):

// assistant.transcription — streaming (isFinal=false) and final
{
  "object": "assistant.transcription",
  "text": "This code is a React hook that...",
  "turn_id": "42",
  "final": true
}

// user.transcription — only final events are displayed
{
  "object": "user.transcription",
  "text": "Can you explain what useMemo does here?",
  "turn_id": "41",
  "final": true
}

The turn_id is used to update streaming entries in-place: the same turn's chunks accumulate in the transcript until final: true arrives.

Stream-Message Chunking (RTC Fallback)

When RTM token generation fails, the ConvoAI agent sends transcripts via the RTC data channel (stream-message event) in a chunked format:

{seq}|{total}|{key}|{payload}

// Example: large message split into 3 chunks:
0|3|msg-abc|{"object":"assistant.transc
1|3|msg-abc|ription","text":"The functi
2|3|msg-abc|on...","turn_id":"5","final":true}

The companion reassembles all chunks keyed by key before parsing JSON.

§16 · Key Design Decisions

App Certificate never leaves the extension host. Tokens are minted in Node.js using agora-token. The companion receives only the signed clientToken, rtmToken, and a pre-computed stopAuth Basic Auth header — never the raw certificate.
Browser companion is a separate HTML file in tmpdir. The VS Code webview sandbox blocks microphone access and direct audio playback. The companion is written to os.tmpdir() and opened via vscode.env.openExternal() to get a full browser context with mic/speaker APIs.
Webview uses RTM only — no RTC join. The sidebar only needs the transcript text, not audio. It joins RTM directly for transcript events without touching the RTC channel, keeping the webview's Agora footprint minimal.
Chat mode reuses the voice ConvoAI agent. idleTimeout:0 prevents auto-stop on silence. session.think() bypasses the STT stage entirely, injecting text directly into the LLM pipeline. The agent still runs TTS but no audio is consumed.
enable_rtm: true is always set. This ensures transcripts are reliably delivered via the RTM channel in both voice and chat mode, regardless of whether the RTC stream-message data channel is available.
Stop endpoint is pre-computed at session start. The browser companion receives stopUrl + stopAuth so it can call the ConvoAI stop endpoint with keepalive: true even when the tab is being closed — ensuring the agent is stopped even if the VS Code extension host is unreachable.

Cost optimization tips

Stop sessions explicitly. ConvoAI charges from session.start() to stop. Voice mode auto-stops after 30 s of silence (idleTimeout:30). Chat mode has idleTimeout:0 — click Stop as soon as you're done.
Use chat for quick questions. Chat mode skips RTC entirely — zero participant-minutes consumed. The ConvoAI cost is identical, but it helps teams stay inside the 10,000 free RTC minutes per month.
One Agora project per team. Each Agora project gets its own free tier (10,000 RTC min, 1M RTM messages). Large teams can spread load across separate projects with distinct appId / appCertificate pairs.
Monitor in Agora Console. The Usage dashboard shows ConvoAI minutes, RTC minutes, and RTM messages in real time. Set billing alerts before you expect to hit thresholds.
